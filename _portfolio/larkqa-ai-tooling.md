At ByteDance LarkQA, I worked on AI engineering systems that automate test case polishing, defect coverage review, and platform testing workflows.

## Overview

This work focused on applying LLMs, RAG, and agent tooling to real QA workflows. The systems cover test case polishing, case execution support, defect coverage review, and reusable platform testing tools.

## My Role

I designed and implemented AI workflow components, asynchronous LLM service calls, RAG recall logic, CLI tools, and Skill-based integrations.

## Key Work

- Built BBC/BBQ test case polishing workflows with **Coze Bot API**, **Aily asynchronous runs**, Fornax GPT, multimodal prompts, and Spring Boot + Redis queues.
- Implemented robust long-chain LLM calls with session creation, message creation, run polling, timeout handling, retry logic, and structured JSON extraction.
- Designed quantitative adoption metrics for polished test steps and scheduled jobs for data-quality maintenance.
- Built RAG-based defect automation coverage review with BBQ/BBC dual recall, multimodal information completeness checks, confidence scoring, and Feishu Bitable output.
- Developed Bits/BOSS testing tools, BOSS CLI, and OpenClaw Skill Gateway for tenant creation, asset unsubscribe, usage query/reporting, and mock-time scenarios.

## Impact

- Reduced manual effort in test case polishing and platform testing.
- Made AI-generated QA artifacts easier to review through structured metrics and traceable outputs.
- Converted repeated testing operations into reusable CLI and Skill tools.

## Tech Stack

**Java, Spring Boot, Python, Node.js, TypeScript, Redis, Coze API, Aily API, Fornax GPT, Viking DB, Milvus, MCP SDK, JWT, Feishu Bitable API**
